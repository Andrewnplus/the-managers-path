                      技術負責人 #

                      對作者而言，Tech Lead 非為特定職級，而是成為資深工程師後都可能承擔的系列職責。他會與被管理人員固定一對一會議、定期給予回饋（職涯發展、推進目標、改進領域）、為其訂定學習領域。這對於晉升都是必要的，因為職位越高階，就越注重領導力和承擔責任的能力。也因此人際技能會比純粹的技術能力更為重要。

                      Tech Lead 入門 #

                      優秀的 Tech Lead 得在事物間取得平衡：有效管理時間，為自己安排適當長度的任務區塊。讓團隊專注在開發工作，幫助其他利害關係人尊重團隊重心，避免增加個人貢獻者的負擔。

                      他的首要任務是用大局觀看待工作，讓專案進度如期推展。這過程中會扮演到以下職責：

                      • 系統架構師和商業分析師：找出需改動的關鍵系統和須打造的關鍵功能，為未來做準備
                      • 專案規劃師：拆分工作為數個可交付部分（Deliverables），讓團隊在同個時段完成最多項目
                      • 軟體開發者和團隊負責人：編寫程式、溝通協調、分派工作，聚焦於不可被妥協的功能上

                      管理專案 #

                      敏捷式開發使我們拆解任務成更小部分，漸進式地交付價值，但面對充滿未知與難掌握的死線時，我們仍須具備專案管理的知識。我們得自發性地主動琢磨專案，對能合理預測和計畫的地方深謀遠慮。過程可能不令人習慣和感到沮喪，卻能讓我們之後有能力建立更大的系統，帶領更大的團隊，因為明白了拆解專案的真義。

                      所謂的專案管理，是將複雜的終極目標拆分成更小部分，用有效地順序排列，並嘗試理出會讓專案進度放慢或完全失敗的未知因素。主要方針如下：

                      • 分解工作：拆解大目標成小任務，並將注意力轉移到工作順序上
                      • 推進細節和未知要素：專案管理的訣竅就是遇到瓶頸或厭倦時也別停下，努力克服未知
                      • 運行時同步調整計畫：規劃的價值在於了解進度，進度不如預期時得讓所有人都知道
                      • 利用規劃所得的洞察來管理需求變更：利用持續更新的已知來判斷變更的成本，得以做出權衡
                      • 臨近專案終點時，重新審視細節：理出「夠好」的界線，讓團隊聚焦於更重要的細節
